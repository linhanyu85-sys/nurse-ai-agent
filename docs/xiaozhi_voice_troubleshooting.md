# 小智语音链路排障（COM5 面包板版）

## 1. 先确认当前模式

当前上位机支持两条链路：

- 串口文字链路：串口收到文本后调用 AI Agent（已可用）
- 设备语音链路：小智固件在设备侧完成唤醒/采音，再通过协议与服务器交互

注意：官方小智固件默认不是“串口收到 `TTS:` 就由板载喇叭播放”。  
如果你看到“本机语音播报完成”，说明是电脑扬声器在播报，这是兜底机制。

## 2. 上位机自检（已内置）

启动 `小智 AI Agent 上位机` 后：

1. 串口选 `COM5`，点击“连接串口”
2. 点击“检查网关”
3. 点击“串口语音自检”
4. 在 15 秒内对设备说“你好小智”，再说一句短指令

判定：

- 通过：日志出现 `xiao_zhi_wake` 或 `xiao_zhi_stt`
- 失败：日志提示“未捕获 wake/stt 事件”，优先排查接线/固件

## 3. 面包板关键引脚（bread-compact-wifi）

麦克风 I2S：

- `WS -> GPIO4`
- `SCK -> GPIO5`
- `SD  -> GPIO6`

喇叭功放 MAX98357：

- `DIN  <- GPIO7`
- `BCLK <- GPIO15`
- `LRC  <- GPIO16`

其余：

- `MIC VDD -> 3V3`
- `MIC GND -> GND`
- `AMP VIN -> 5V`
- `AMP GND -> GND`

## 4. 现场高频故障

- 麦克风模块焊接方向与接线图不一致（“图标朝上/朝下”两套图不能混用）
- 刷错固件（板型不匹配会导致麦克风/喇叭无效）
- 功放接线或供电缺失（能识别但无声）
- 串口占用（两个程序同时占用 `COM5`）

## 5. 最稳操作顺序（不伤板）

1. 只保留一个上位机进程
2. 断电后逐线对照 `第一步接线图` 检查
3. 上电后做“串口语音自检”
4. 若仍失败，再考虑重新烧录匹配板型的固件
